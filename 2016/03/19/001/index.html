记录用hexo搭建Blog过程中遇到的问题

Mar 19, 2016

1、关于生成并推送到github

第一步： 配置根目录的_config.yml文件。

deploy: 
    type: git
    repository: https://github.com/LastMagician/LastMagician.github.io.git repository—name 
    branch: master

注：从hexo 3.0开始，type要改成git才能正常deploy

第二步： 生成静态文件并推送到github，命令如下：

$ hexo g  # 生成静态文件
$ hexo d  # 部署网站
或
$ hexo g --d  # 完成后部署

注：完成第一步的配置之后，第二步部署网站的时候会提示输入github的用户名和密码

  • 若执行成功，会自动将生成的静态文件同步到github
  • 报报错： error：deoloyer not found github，执行如下命令：
npm install hexo-deployer-git --save
  • 上一步成功之后，再次执行：
hexo d

2、进入主页文章列表直接显示全部，而不是显示一部分摘要或者more

解决办法：在文章中加一句 <!--more-->，在这个标记之前的就是摘要。
效果：http://lastmagician.github.io/

3、push到github时重复输入用户名和密码

问题原因及解决方案：

主要的原因是采用了 https://github.com 类型的repository，如下：

deploy:
  type: git
  repository: https://github.com/LastMagician/LastMagician.github.io.git
  branch: master

解决方案，改成ssh类型的repository，如下

deploy:
  type: git
  repository: git@github.com:LastMagician/LastMagician.github.io.git
  branch: master

注意 repository 的值的变化

4、生成推送到github时报错：ERROR Deployer not found: git

解决方法： 安装 hexo-deployer-git，命令如下

npm install hexo-deployer-git –save
